## 6. VALUE ASSESSMENT FRAMEWORK

### 12-Dimension Scoring (each 0-5)

#### Positive Dimensions (8) — What value does this create?

| Dimension | Weight | What It Measures |
|-----------|--------|-----------------|
| revenue_impact | **1.5x** | Direct or indirect income effect |
| cost_efficiency | **1.2x** | Savings or better resource deployment |
| time_leverage | **1.3x** | How much faster value is realized |
| strategic_alignment | **2.0x** | Fit with doctrine and long-term direction |
| customer_human_benefit | **1.4x** | Improves lived experience or trust |
| knowledge_asset_creation | **1.1x** | Produces reusable intelligence |
| compounding_potential | **1.8x** | Creates future flywheel effects |
| reversibility | **1.0x** | Can be undone cheaply if wrong |

#### Penalty Dimensions (4) — What risk/cost does this carry?

| Dimension | Weight | What It Measures |
|-----------|--------|-----------------|
| downside_risk | **2.0x** | Magnitude of harm if wrong |
| execution_drag | **1.2x** | Coordination burden / friction |
| uncertainty | **1.5x** | Unknowns not yet bounded |
| ethical_misalignment | **3.0x** | Conflict with doctrine or values |

### Scoring Formulas

```
Gross Value     = sum(positive dimensions)                    // Range: 0-40
Penalty         = sum(penalty dimensions)                     // Range: 0-20
Net Value       = Gross - Penalty                             // Range: -20 to 40

Weighted Gross  = sum(dimension × weight) for positives       // Range: 0-57.3
Weighted Penalty = sum(dimension × weight) for penalties      // Range: 0-38.5
Weighted Net    = Weighted Gross - Weighted Penalty            // Range: -38.5 to 57.3
```

### Value Classification

|---------------|-------------------|---------|
| **strong_candidate** | ≥ 20 | High value — proceed to trust assessment |
| **conditional_candidate** | 12-19 | Moderate value — proceed with caution |
| **weak_candidate** | 0-11 | Low value — likely escalate or block |
| **block** | < 0 | Negative value — block unless exceptional reason |

## 7. TRUST ASSESSMENT FRAMEWORK

### 7-Dimension Scoring (each 0-5)

| Dimension | What It Measures | Promotion Guard? |
|-----------|-----------------|-----------------|
| evidence_quality | Source quality, freshness, completeness | **T2: ≥2, T3: ≥3, T4: ≥4** |
| logic_integrity | Internal consistency and first-principles fit | **T3: ≥3, T4: ≥4** |
| outcome_history | Prior success in similar contexts | **T4: ≥4** |
| context_fit | Relevance to current environment | — |
| stakeholder_clarity | Known owners, incentives, constraints | — |
| risk_containment | Ability to bound downside | **T3: ≥3, T4: ≥4** |
| auditability | Can rationale be reviewed later | **T2: ≥2, T3: ≥3, T4: ≥4** |

### Trust Tier Calculation

| Tier | Total Score Range | Avg per Dimension | Multiplier | Meaning |
|------|------------------|-------------------|-----------|---------|
| **T0** Unqualified | < 10 | < 1.4 | 0.2 | Insufficient basis |
| **T1** Observed | 10-16 | 1.4-2.3 | 0.5 | Signal exists, not ready |
| **T2** Qualified | 17-23 | 2.4-3.3 | 0.8 | Enough to recommend |
| **T3** Certified | 24-29 | 3.4-4.1 | 1.0 | Execute with bounded autonomy |
| **T4** Delegated | 30-35 | 4.3-5.0 | 1.2 | Automate recurring class |

### Promotion Guards (Cannot Skip)

**To reach T2:** `evidence_quality ≥ 2` AND `auditability ≥ 2`
**To reach T3:** `evidence_quality ≥ 3` AND `logic_integrity ≥ 3` AND `risk_containment ≥ 3` AND `auditability ≥ 3`
**To reach T4:** `evidence_quality ≥ 4` AND `logic_integrity ≥ 4` AND `outcome_history ≥ 4` AND `risk_containment ≥ 4` AND `auditability ≥ 4`

### Demotion Triggers (Automatic)
- Outcome variance is high
- Evidence becomes stale
- Assumptions fail
- Environment materially changes
- Audit gap appears

### Weighted Trust Calculation

```
trust_total   = sum(7 dimensions)             // Range: 0-35
trust_average = trust_total / 7               // Range: 0-5
adjusted_trust = trust_average × tier_mult    // Used in priority scoring
```

## 8. RTQL — RECURSIVE TRUST QUALIFICATION LOOP

### Purpose
Classifies the trust quality of information inputs through an 11-checkpoint gate sequence before they can influence decisions.

### 9 Trust Stages (Progressive)

| Stage | Multiplier | Write Target | Meaning |
|-------|-----------|-------------|---------|
| **noise** | 0.00 | quarantine | No signal — discard |
| **weak_signal** | 0.35 | staging | Low integrity/exposure |
| **echo_signal** | 0.50 | staging | May reflect bias/echo chamber |
| **qualified** | 1.00 | candidate_registry | Passes qualification gate |
| **certification_gap** | 0.85 | candidate_registry | Qualified but not certifiable yet |
| **certified** | 1.15 | operational_registry | Full certification passed |
| **research_grade** | 1.30 | insight_registry | Certified + novel yield |
| **first_principles_candidate** | 1.50 | principles_registry | All causal checks pass |
| **axiom_candidate** | 1.50-2.00 | axiom_review_queue | Governance-approved axiom |

### Gate Sequence

```
INPUT
  │
  ▼
Gate 0: Identity & Provenance ─── FAIL → NOISE (quarantine)
  │ PASS
  ▼
Gate 1: Source Integrity ≥ 4 ──┐
Gate 2: Exposure Count ≥ 3 ────┤── ANY FAIL → WEAK_SIGNAL or ECHO_SIGNAL
Gate 3: Independence ≥ 4 ──────┘
  │ ALL PASS (QUALIFIED)
  ▼
Gate 4: Explainability ≥ 6 ───┐
Gate 5: Replicability ≥ 6 ────┤── ANY FAIL → CERTIFICATION_GAP
Gate 6: Adversarial Robust ≥ 6┘
  │ ALL PASS (CERTIFIED)
  ▼
Gate 7: Novelty Yield ≥ 6 ────── FAIL → CERTIFIED (no novelty)
  │ PASS
  ▼
Gate 8:  Reveals Causal Mechanism ──┐
Gate 9:  Is Irreducible ────────────┤── ANY FAIL → RESEARCH_GRADE
Gate 10: Survives Authority Removal ┤
Gate 11: Survives Context Shift ────┘
  │ ALL PASS
  ▼
FIRST_PRINCIPLES_CANDIDATE
  │
  ▼ (governance promotion only)
AXIOM_CANDIDATE
```

### Research Actions Generated on Failure

| Failed Gate | Research Action |
|------------|----------------|
| source_integrity < 4 | "Locate or establish auditable source trail" |
| exposure_count < 3 | "Monitor for repeat appearances across independent contexts" |
| independence < 4 | "Seek cross-domain or independent confirmation" |
| explainability < 6 | "Decompose claim into causal variables and mechanism" |
| replicability < 6 | "Define and execute a replication protocol" |
| adversarial_robustness < 6 | "Design adversarial/red-team test for this claim" |
| novelty_yield < 6 | "Assess whether input produces materially new information" |
| causal mechanism = false | "Identify underlying causal mechanism" |
| irreducible = false | "Test irreducibility — can this be decomposed further?" |
| authority removal = false | "Verify claim holds without authority backing" |
| context shift = false | "Test claim across different operational contexts" |

### Pipeline Integration
- **rtql_prefilter_passes():** Returns `True` if stage ≥ ECHO_SIGNAL. NOISE and WEAK_SIGNAL are blocked.
- RTQL trust_multiplier adjusts value scoring: `adjusted_value = raw_value × rtql_multiplier`
- Low RTQL stage proceeds with degraded multiplier + warning audit entry

## 9. CERTIFICATE CHAIN LOGIC

### Chain Sequence: QC → VC → TC → EC

**Hard Rule:** No EC can be issued without valid QC + VC + TC.

| Certificate | Prerequisites | Expiry | Requirements |
|------------|--------------|--------|-------------|
| **QC** (Qualified Context) | None (first) | 30 days | title, owner, problem_statement, requested_action, stakeholders — all non-empty |
| **VC** (Value Confirmed) | QC valid | 14 days | QC valid + `gross_value > 0` + `strategic_alignment ≠ 0` + scores validate |
| **TC** (Trust Certified) | VC valid | 14 days | VC valid + `trust_tier ≥ T2` + evidence_refs non-empty + assumptions listed (D3-D6) |
| **EC** (Execution Cleared) | TC valid | 7 days | TC valid + all 7 gates pass + execution_plan + monitoring_metric + rollback_trigger (non-D0) |

### Chain Builder Flow
```
Issue QC → VALID? ──NO──→ return partial chain (QC denied)
  │ YES
Issue VC → VALID? ──NO──→ return partial chain (VC denied)
  │ YES
Issue TC → VALID? ──NO──→ return partial chain (TC denied)
  │ YES
Issue EC → VALID? ──NO──→ return partial chain (EC denied)
  │ YES
CHAIN COMPLETE ✓
```

## 13. PRIORITY SCORING FORMULA

### Formula

```
Priority Score = Base × Time_Leverage × Strategic_Alignment × Ethos_Alignment × Trust_Multiplier × RTQL_Multiplier
```

Where:
```
Base = (Expected_Value - Total_Cost - Risk_Penalty) × Probability_of_Success

Expected_Value     = gross_value (sum of 8 positive dimensions)
Total_Cost         = execution_drag
Risk_Penalty       = downside_risk + uncertainty
Probability        = 0.7 (default)

Time_Leverage      = max(0.1, dimension / 5.0)
Strategic_Alignment = max(0.1, dimension / 5.0)
Ethos_Alignment    = max(0.1, alignment_composite)

Trust_Multiplier   = {T0: 0.2, T1: 0.5, T2: 0.8, T3: 1.0, T4: 1.2}
RTQL_Multiplier    = {noise: 0.0, weak: 0.35, echo: 0.5, qualified: 1.0, ...up to 1.5}
```

### Safeguards
- Base floor: -10 (prevents extreme negative inversion)
- All normalization floors: 0.1 (prevents zero-out)
- Result rounded to 3 decimal places

## 15. ADAPTIVE LEARNING SYSTEM

### Learning Value Index (LVI) — Organizational Maturity

| Stage | LVI Range | Min Decisions | Min Accuracy |
|-------|----------|---------------|-------------|
| **Decision Recording** | 10-20 | 10 | 0% |
| **Pattern Identification** | 20-35 | 10 | 70% |
| **Causal Template** | 35-55 | 20 | 75% |
| **Predictive Refinement** | 55-75 | 50 | 85% |
| **Org Capability** | 75-90 | 100 | 90% |

### 4 Calibration Engines

**Engine 1: Causal Model Calibration**
- Input: Predicted vs actual outcomes per decision type
- Accuracy = actual / predicted
- If avg_accuracy > 0.85 → increase weight by 5%
- If avg_accuracy < 0.60 → decrease weight by 10%
- Recalibration trigger: overall accuracy < 0.70

**Engine 2: OVS Projection Refinement**
- Input: Predicted vs actual OVS scores
- Calculates bias correction and absolute error
- Recalibration trigger: avg absolute error > 5.0 points

**Engine 3: Decision Velocity Benchmarking**
- Target decision time: 48 hours
- Target escalation time: 24 hours
- Recalibration trigger: avg decision time > 72 hours (1.5× target)

**Engine 4: Authority Calibration**
- Monitors escalation accuracy and false alarm rate
- If escalation_accuracy < 85% → tighten thresholds by 5%
- If false_alarm_rate > 10% → tighten thresholds by 8%

### Core Learning Rule
> A repeated decision class should get **easier, faster, and safer** over time.

## 16. OUTCOME VARIANCE & TRUST FEEDBACK

### Variance Calculation

| Dimension | Tolerance | POSITIVE | NEGATIVE |
|-----------|-----------|----------|----------|
| **Value** | ±10% | actual > expected + 10% | actual < expected - 10% |
| **Timeline** | ±3 days | faster by > 3 days | slower by > 3 days |
| **Risk** | — | no surprise | risk materialized unexpectedly |

### Composite Variance Score (-1.0 to +1.0)

```
Composite = (Value × 0.50) + (Timeline × 0.30) + (Risk × 0.20)

Direction scores: POSITIVE = +1.0, NEUTRAL = 0.0, NEGATIVE = -1.0
Risk component: no surprise = +0.5, materialized (expected) = -0.3, surprise = -1.0
```

### Trust Recommendation Derivation

| Composite Score | Recommendation | Action |

## 21. HMAC TRUST CERTIFICATES

### Security Model
- HMAC-SHA256 signed with secret key
- Each certificate has corresponding `.md` file on disk
- Engine refuses to load certificate if: signature invalid, expired, MD file missing, or MD file tampered

### 4-Point Validation (Every Lookup)
1. Valid HMAC signature (cryptographic integrity)
2. Not expired
3. MD file present on disk (audit trail exists)
4. MD file signature matches cert (no tampering)

## 24. OUTCOME ESTIMATION MODEL

### How the Engine Estimates Outcome Probability

The engine does not produce a single "% chance" number but builds a **multi-dimensional confidence envelope** that converges on an outcome estimate:

```
Outcome Confidence = f(Value Strength, Trust Quality, RTQL Stage, Gate Results, Historical Accuracy)
```

#### Component Contributions to Outcome Confidence

| Factor | Weight in Estimation | Signal |
|--------|---------------------|--------|
| **Value Classification** | 25% | strong_candidate (≥20) → high confidence in value delivery |
| **Trust Tier** | 25% | T3/T4 → historically validated decision framework |
| **RTQL Stage** | 15% | certified+ → information quality is high |
| **Gate Pass Rate** | 15% | 7/7 gates → all safety checks satisfied |
| **Certificate Chain** | 10% | Complete → every prerequisite verified |
| **Historical Accuracy** | 10% | Learning loop feedback from past decisions of same class |

#### Estimated Outcome by Verdict

| Verdict | Confidence Basis | Estimated Success Rate | Why |
|---------|-----------------|----------------------|-----|
| **AUTO_EXECUTE** | All gates pass, T3+, R1/R2, chain complete | **85-95%** | Maximum validation passed. Historical decisions at this tier succeed 85%+ per learning loop calibration. Residual 5-15% risk from unknown-unknowns and environment change. |
| **ESCALATE_TIER_1** | Minor gate failures, recoverable | **70-85%** | Core value and trust are sound but one safety check needs human judgment. Human review catches the gap ~80% of the time. |
| **ESCALATE_TIER_2** | Significant gate failures or high-class decision | **55-75%** | Decision requires executive judgment. Value proposition exists but risk/authority/approval gaps create uncertainty. Success depends on human decision quality. |
| **ESCALATE_TIER_3** | D5/D6, major blockers | **40-65%** | High-stakes decision with significant unknowns. Board-level review needed. Wide confidence interval reflects genuine uncertainty. |
| **BLOCK** | Multiple gates fail, ethical/trust issues | **< 30%** | Fundamental gaps in value, trust, or ethics. Decision should not proceed in current form. Blocking prevents negative outcome. |
| **NEEDS_DATA** | Data gaps prevent assessment | **Indeterminate** | Cannot estimate — insufficient information to form a basis. Correct action is to gather data before estimating. |

#### Why These Ranges

1. **AUTO_EXECUTE (85-95%):** The engine requires ALL of: complete certificate chain, 7/7 gate passes, T3+ trust (which requires promotion guards including outcome_history), R1/R2 reversibility, and value above class threshold. Each condition independently screens for a failure mode. Combined, they represent the highest-confidence decisions the system can produce. The 5-15% residual comes from Principle #15 ("the map is not the territory") — models always have residual uncertainty.

2. **ESCALATE_TIER_1 (70-85%):** One or two gates failed, but the fundamental value/trust structure is intact. Human review at this tier catches ~80% of the gap (per Engine 4: Authority Calibration which targets 85% escalation accuracy). The lower bound accounts for cases where the human reviewer lacks context.

3. **ESCALATE_TIER_2 (55-75%):** The decision has genuine complexity — financial impact (D3), strategic direction (D4), or significant gate failures. Executive judgment adds value but also introduces human bias. Wider range reflects Principle #7 (risk is asymmetrical).

4. **ESCALATE_TIER_3 (40-65%):** D5/D6 decisions are inherently high-uncertainty. Legal, ethical, or irreversible actions have tail risks that models cannot fully capture. The wide range reflects Principle #3 (all resources are finite, including predictive accuracy for novel situations).

5. **BLOCK (< 30%):** Multiple simultaneous gate failures indicate systemic misalignment between the proposed action and the engine's validation framework. Proceeding would violate Principle #1 (reality before preference) and Principle #14 (ethics are a constraint).

#### How Learning Refines These Estimates Over Time

```
CYCLE: Execute → Record Outcome → Calculate Variance → Derive Trust Recommendation → Update Weights → Re-estimate

LVI Stage 1 (10-20): Recording only — estimates are default ranges above
LVI Stage 2 (20-35): Patterns emerge — ranges narrow by ~5%
LVI Stage 3 (35-55): Causal templates — class-specific accuracy improves
LVI Stage 4 (55-75): Predictive refinement — ranges narrow by ~10%
LVI Stage 5 (75-90): Org capability — ranges narrow by ~15%, near-deterministic for recurring classes
```
